# 建具表 OCR（AI画像読み取り）セットアップ手順

設計仕様詳細の画像から、WD番号・種類・幅・高さを **AI（Claude）が自動で読み取って** 建具リストに入れる機能です。

## 必要なもの
- Python（このPCに入っています）
- Anthropic の APIキー（有料・従量課金。画像1枚あたり数円程度）

---

## 初回だけの準備（3ステップ）

### 1. APIキーを用意する
- https://console.anthropic.com/ にログイン → APIキーを発行（`sk-ant-...` で始まる文字列）

### 2. .env ファイルにキーを設定する
- このフォルダの **`.env.example`** をコピーして、名前を **`.env`** に変更
- `.env` をメモ帳で開き、次の行のキー部分を自分のキーに書き換えて保存
  ```
  ANTHROPIC_API_KEY=ここにあなたのAPIキーを貼り付け
  ```
- ⚠ `.env` は**他人に渡さない・共有しない**でください（キーは秘密情報です）

### 3. （自動）ライブラリのインストール
- 次の「使うとき」で `OCR起動.bat` を初めて実行すると自動で入ります（手動なら `pip install anthropic`）

---

## 使うとき（毎回）

1. **`OCR起動.bat` をダブルクリック** → 黒い画面が出て「サーバー起動」と表示されたらOK（この画面は開いたままにする）
2. ブラウザで `仕様一覧表メーカー.html` を開く → 「建具表」タブ → 「設計仕様詳細」
3. 「📷 画像を選択」で設計仕様詳細の画像を取り込む
4. 「📐 画像から W・H を自動入力」を押す → AIが読み取って建具リストに反映
5. 終わったら黒い画面で **Ctrl + C**（またはウィンドウを閉じる）でサーバー停止

> サーバーを起動していない場合は、自動で「簡易OCR（端末内）」に切り替わります。精度は落ちます。

---

## よくある質問

- **黒い画面に「ANTHROPIC_API_KEY が未設定」と出る** → `.env` のキー設定を見直してください（手順2）。
- **「読み取りに失敗」と出る** → ネット接続、キーの有効性、画像が鮮明かを確認。
- **費用が気になる** → 使うときだけ `OCR起動.bat` を起動すればOK。起動していなければ課金されません。
- **モデルを変えたい** → `.env` に `OCR_MODEL=claude-sonnet-4-6` を追記すると安価なモデルに変更できます（精度はやや下がります）。

## しくみ（技術メモ）
- `ocr_server.py` が `127.0.0.1:8788` で待ち受け、画像を Claude のビジョンAPIに送って構造化JSONで受け取る。
- APIキーは **HTMLには書かず**、サーバー側の `.env`（環境変数）から読み込む。
- アプリの「📐」ボタンはまずこのサーバーを呼び、未起動なら端末内OCR（Tesseract）に自動フォールバック。
